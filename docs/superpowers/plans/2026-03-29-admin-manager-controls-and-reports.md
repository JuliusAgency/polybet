# Admin Manager Controls And Reports Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement immediate blocking logout semantics, clear manager/user separation in admin profile UI, hierarchical bet stake limits, and super-admin PDF report exports.

**Architecture:** Core business enforcement stays in Postgres RPCs (authoritative). Frontend consumes computed limit metadata and exposes admin controls. PDF exports are generated by a dedicated Supabase Edge Function with strict super-admin authorization and audited export actions.

**Tech Stack:** React + TypeScript, TanStack Query, Supabase Postgres migrations/RPCs, Supabase Edge Functions (Deno), i18next, tsx test runner.

---

## Scope Decomposition

Spec includes four subsystems. Implementation will be split into four deliverable tracks with independent verification:

1. Auth revocation hardening (blocking/logout)
2. Limit model and enforcement (DB + admin UI)
3. Manager profile IA refactor (UX clarity)
4. PDF export/reporting pipeline (backend + admin UI)

Each track is testable independently and can be committed separately.
